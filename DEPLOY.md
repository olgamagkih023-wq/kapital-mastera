# Капитал Мастера — инструкция по развёртыванию

Этот проект — статическое PWA + serverless-функции на Vercel.
Бэкенд: Firebase Realtime Database (данные) + Firebase Auth (вход).

## 1. Переменные окружения в Vercel
Vercel → проект → Settings → Environment Variables. Добавить:

- `TG_BOT_TOKEN`   — токен Telegram-бота (для уведомлений мастеру)
- `TG_CHAT_ID`     — chat_id, куда слать уведомления
- `DEEPSEEK_API_KEY` — ключ DeepSeek (для чат-бота-помощника)

Без них функции /api/notify и /api/chat вернут 500.

## 2. Firebase: включить вход по Email/Password
Консоль Firebase → Authentication → Sign-in method →
Email/Password → Enable → Save.

Вход в приложении сделан по схеме «телефон как логин»:
номер превращается в технический email вида 79991234567@kapital.local.

## 3. Firebase: опубликовать правила безопасности
Консоль Firebase → Realtime Database → Rules →
вставить содержимое database.rules.json → Publish.

ВАЖНО: файл database.rules.json в репозитории сам по себе ничего не меняет.
Правила вступают в силу ТОЛЬКО после Publish в консоли (или `firebase deploy`).
Публикуйте строгие правила ПОСЛЕ того, как убедитесь, что вход
по Email/Password работает (иначе приложение потеряет доступ к данным).

## 4. Домены для защиты API
В api/_guard.js, массив ALLOWED_HOSTS — добавьте свой кастомный домен,
когда подключите его. Иначе запросы с него получат 403.
Превью-деплои *.vercel.app того же проекта уже разрешены.

## 5. Обновление приложения (service worker)
При КАЖДОМ изменении index.html / кода поднимайте версию кэша
в app/sw.js: CACHE_NAME = 'kapital-mastera-v3' и т.д.
Иначе у пользователей останется старая версия из кэша.

## Что осталось открытым (известные компромиссы)
- Публичная страница записи (booking.html) читает ветку masters целиком —
  так устроена проверка статуса и отмена записи клиентом по ID.
  Это видимость данных записей, не финансов. Закрыть можно, только
  переписав логику booking на серверную функцию.
- Анонимное создание заявок не имеет капчи — возможен спам заявками.
  При необходимости добавить reCAPTCHA или запись через /api.
- Rate-limiting на /api — in-memory, защищает от примитивного спама,
  не от распределённой атаки. Для надёжности — Vercel KV / Upstash Redis.
